# Instructions for Pushing Kubernetes Files to GitHub

Follow these steps to push only the Kubernetes (k8s) configuration files to your GitHub repository:

## Prerequisites
- Git installed and configured
- Access to your GitHub repository

## Steps

1. **Navigate to your project directory**
   ```powershell
   cd "c:\Users\AdMin\Documents\ingenerie-logiciel\master\2s\richdalestage\Richdale_Internship_May17"
   ```

2. **Create a new branch for the Kubernetes files**
   ```powershell
   git checkout -b k8s-config
   ```

3. **Add only the k8s directory files to the staging area**
   ```powershell
   git add k8s/
   ```

4. **Commit the changes with a meaningful message**
   ```powershell
   git commit -m "Add Kubernetes configuration files for image classifier deployment"
   ```

5. **Push the branch to GitHub**
   ```powershell
   git push -u origin k8s-config
   ```

   If you encounter authentication issues, try:
   ```powershell
   # Using GitHub CLI
   gh auth login
   
   # Or setting up credentials
   git config --global user.name "Your GitHub Username"
   git config --global user.email "your.email@example.com"
   ```

6. **Create a Pull Request (optional)**
   - Go to your GitHub repository
   - Navigate to "Pull Requests"
   - Click "New Pull Request"
   - Select base branch (usually main) and compare branch (k8s-config)
   - Create the pull request

## Alternative: Push Only Selected Files to Existing Branch

If you prefer to push the k8s files to the main branch:

1. **Checkout the main branch**
   ```powershell
   git checkout main
   ```

2. **Add only the k8s files**
   ```powershell
   git add k8s/deployment.yaml k8s/service.yaml k8s/hpa.yaml
   ```

3. **Commit and push**
   ```powershell
   git commit -m "Add Kubernetes configuration files"
   git push origin main
   ```

## Using GitHub Desktop (Alternative)

If you're having issues with the command line:

1. Open GitHub Desktop
2. Add your repository
3. Select only the k8s files in the changes tab
4. Commit with a message
5. Push to GitHub

## File Descriptions

These are the files you're pushing:

1. **deployment.yaml**: Kubernetes Deployment configuration for the image classifier
2. **service.yaml**: Kubernetes Service to expose the application
3. **hpa.yaml**: Horizontal Pod Autoscaler for automatic scaling based on CPU usage
